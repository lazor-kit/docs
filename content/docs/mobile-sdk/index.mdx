# LazorKit Wallet Mobile Adapter

The LazorKit Wallet Mobile Adapter is a React Native SDK for integrating LazorKit smart wallets. It handles WebAuthn (Passkey) authentication and Solana blockchain interactions on mobile devices.

## System Overview

The Mobile SDK bridges the gap between React Native and native mobile APIs. It manages the lifecycle of a WebAuthn credential, from creation to assertion, and coordinates with the Solana blockchain.

- **Secure Storage**: Private keys are generated within the device's Trusted Execution Environment (TEE)â€”Secure Enclave on iOS and Titan M on Android. They are never exposed to the application layer.
- **Session Management**: Connection state is persisted using `AsyncStorage`, allowing the wallet to remain connected across app restarts without repeated biometric prompts until a signing action is required.
- **Native Modules**: The SDK uses native platform bindings to invoke the system's passkey dialogs, ensuring a familiar and secure UI for the user.

## Installation

```bash
npm install @lazorkit/wallet-mobile-adapter
```

## Integration Flow

1.  **Initialization**: App initializes `LazorKitProvider`.
2.  **Check State**: SDK checks `AsyncStorage` for existing session.
3.  **Connect/Create**:
    -   If no session: Triggers WebAuthn registration (Passkey creation).
    -   If session exists: Triggers WebAuthn assertion (Biometric auth).
4.  **Smart Wallet**:
    -   New User: Deploys PDA for the smart wallet.
    -   Existing User: Fetches on-chain account state.
5.  **Signing**: User approves action via biometrics; SDK submits transaction to Solana.
## Architecture Components

The SDK follows a layered architecture:

### 1. React Layer (`src/react/`)
- **`LazorKitProvider`**: Context provider that initializes the `useWalletStore`.
- **`useLazorWallet`**: Hook exposing wallet state (`isConnected`, `isSigning`) and methods (`connect`, `signMessage`).

### 2. Core Layer (`src/core/`)
- **`useWalletStore`**: Zustand store with `AsyncStorage` persistence. Manages the wallet session.
- **`actions.ts`**: Implements business logic for connection and signing flows.

### 3. Contract Layer (`src/contract-integration/`)
- **`LazorkitClient`**: Wrapper around Anchor for interacting with the LazorKit on-chain program.
- **`types.ts`**: Defines `SmartWalletActionArgs`, `WalletState`, and `PasskeySignature`.

## WebAuthn Integration

The SDK does **not** handle WebAuthn directly. It delegates to the **LazorKit Portal** via `expo-web-browser`:

1.  **Connect**: Opens `${ipfsUrl}/connect`. The portal handles the `navigator.credentials.create()` call.
2.  **Sign**: Opens `${ipfsUrl}/sign`. The portal handles the `navigator.credentials.get()` call.
3.  **Callback**: The portal redirects back to the app with the `credentialId` or `signature` in the URL parameters.
